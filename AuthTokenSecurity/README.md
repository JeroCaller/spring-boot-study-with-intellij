# 개요

- 이 예제 프로젝트에서는 토큰 인증 방식에서 RTR(Refresh Token Rotation) 및 Token Reuse Detection(TRD)에 대한 이해를 위해 이를 구현합니다.

# 구현사항

- 현재 유효한 refresh token과 바로 직전에 만료된 refresh token을 DB에 저장.
- access token 만료 시 이를 갱신할 수 있는 endpoint 구현.
  - 갱신 시 access token 뿐만 아니라 refresh token도 새로 발급.
  - 이 과정에서 새로 발급된 refresh token을 DB에 기록, 바로 이전 refresh token은 만료된 토큰으로, 별도의 컬럼에 기록.
- 사용자(user) 회원가입, 조회, 수정, 탈퇴 기능(CRUD) 구현.
- 사용자별 보호된 리소스에 대한 CRUD 기능 구현.
  - 이 때 사용자 자신이 보유한 리소스에 대해서만 CRUD 작업 가능. 
  - 다른 사용자의 보호된 리소스에 접근조차 하지 못하도록 구현.
- 이미 만료된 refresh token으로의 요청이 들어올 경우 이를 감지하고, 이에 대한 반응을 REST json으로 응답하도록 구현.

# 테스트 주요 요구사항

- RTR 및 TRD 테스트
  - 인증받지 않은 가상의 해커를 생성하고, 이 해커가 인증된 사용자의 인증용 토큰을 탈취했다고 가정.
  - 인증받지 않은 가상의 해커가 인증된 사용자의 인증용 토큰 탈취 후, 보호된 리소스 접근 요청 시 이를 막고, 이상 감지 가능한지 테스트.
    - 사용자가 먼저 토큰 갱신을 요청할 때와, 해커가 먼저 토큰 갱신을 요청할 때의 시나리오로 나눠 테스트.
